# Contributing Guidelines\n\nThank you for your interest in improving this project!\n\n## How to Report Bugs\n\n1. **Open an Issue** - Click [here](../../issues)\n2. **Describe the problem clearly** with OS, Python version, and steps to reproduce\n3. **Include error messages** and relevant code snippets\n4. **Be patient** - maintainers review issues as time permits\n\n## How to Suggest Features\n\n1. **Start a Discussion** - Click [here](../../discussions) first\n2. **Explain the use case** and why you think it's useful\n3. **Provide examples** of how it would be used\n\n## Pull Request Process\n\n1. **Fork** the repository\n2. **Clone** your fork:\n   ```bash\n   git clone https://github.com/YOUR-USERNAME/rover-wheel-compariso.git\n   cd rover-wheel-compariso\n   ```\n3. **Create a branch** for your changes:\n   ```bash\n   git checkout -b feature/your-feature-name\n   ```\n4. **Make your changes** and commit:\n   ```bash\n   git add .\n   git commit -m \"Describe your change\"\n   ```\n5. **Push** your branch:\n   ```bash\n   git push origin feature/your-feature-name\n   ```\n6. **Open a Pull Request** and wait for review\n\n## Code Style\n\n- Python: PEP 8\n- Arduino/C: K&R or Google C++ style\n- Comments: Thai or English (be clear)\n\n## Testing\n\n- Test your changes before submitting\n- Make sure you don't break existing functionality\n- Include tests if possible\n\n## Documentation\n\n- Update README if needed\n- Add comments to complex code\n- Include usage examples\n\n## Code of Conduct\n\nBe respectful, open-minded, and cooperative.\n\n---\n\nThank you!
